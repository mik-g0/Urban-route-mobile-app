# Urban-route-mobile-app


Инструкция по сборке и установке APK через командную строку для проекта Android:

1. Перейди в папку проекта.

2. Выполни команду Gradle для сборки отладочной версии:

~~~
gradlew assembleDebug
~~~

3.Установи APK на устройство или эмулятор:

 Убедись, что adb (Android Debug Bridge) доступен. Обычно он находится в папке platform-tools SDK.
~~~
Полный путь к adb.exe на Windows может быть таким:

C:\Users\<имя пользователя>\AppData\Local\Android\Sdk\platform-tools\adb.exe
~~~
4. Установи APK на устройство или эмулятор:
~~~
C:\Users\<имя пользователя>\AppData\Local\Android\Sdk\platform-tools\adb.exe install -r app\build\outputs\apk\debug\app-debug.apk
~~~
Если установка прошла успешно, увидишь сообщение:
~~~
Performing Streamed Install
Success
~~~
5. Запусти приложение

Теперь приложение можно открыть на телефоне или эмуляторе через меню приложений.

После успешной сборки APK будет находиться в папке:
~~~
 app/build/outputs/apk/debug/app-debug.apk
~~~

## План монетизации и публикации приложения

### Монетизация приложения:

1. Реклама:

   - Интеграция Google AdMob для отображения баннеров или межстраничных объявлений.

   - Реклама может показываться при открытии карты или при просмотре маршрутов.

2. Премиум-функции:

   - Подписка или единовременная покупка, позволяющая:

       - Убирать рекламу.

       - Сохранять больше маршрутов.

       - Получать доступ к эксклюзивным маршрутам или дополнительной информации о городах.

3. Партнёрские ссылки (по желанию):

   - Возможность интеграции ссылок на сервисы бронирования или туристические ресурсы, если приложение показывает достопримечательности.

--------------------

### Публикация приложения:

1. Регистрация аккаунта разработчика в Google Play Console.

2. Подготовка материалов для публикации:

   - Значки, скриншоты, видео демонстрации приложения.

   - Политика конфиденциальности (при работе с пользовательскими данными).

## Средства обеспечения безопасности приложения

В данном проекте применены следующие меры безопасности:

1. Хранение данных:

   - Локальная база данных Room используется для хранения маршрутов и сохранённых маршрутов.

   - Данные пользователей не отправляются на внешние серверы, что защищает конфиденциальность.

2. Сетевые запросы:

   - Все HTTP-запросы выполняются через OkHttp с HTTPS, что обеспечивает шифрование данных.

   - Настроены таймауты (callTimeout, connectTimeout, readTimeout) для предотвращения зависаний и перегрузки сети.

   - Исключения обрабатываются с помощью try/catch, что защищает приложение от падений при ошибках сети или некорректных данных.

3. Безопасность интерфейса:

   - Все операции с базой данных выполняются через ViewModel с использованием viewModelScope.launch, что предотвращает одновременные неконтролируемые изменения данных.

   - Обработка nullable значений предотвращает ошибки и падение приложения при отсутствии данных.

4. Конфиденциальность и защита ключей:

   - На текущий момент приложение не хранит API-ключи в коде.

   - Данные о выбранных городах и маршрутах остаются локальными на устройстве пользователя.
